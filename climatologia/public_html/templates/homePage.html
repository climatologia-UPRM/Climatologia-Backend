Climatology's API is based around REST webservices for general operations. This API sends real time weather data and events.

Base URL
		http://climatologia.uprm.edu:8008/api

This url has possible inputs of:

		http://climatologia.uprm.edu:8008/api?q=xxx[&additional partameters]

where "q" has default value of null but can be equal to:

  • station
  • data
Note: Additional parameters depend on the query

Query Use

Each quary has its own parameters that can be used to filter what type of information you want to request.

Station:

In this query we have a default value of all. This means that it will return a list of every station in json format. To help filter out what type of station we want to be requested, we use the type parameter. This is then followed with the station that you wish to request.

Example:
		  "type=" + stationtype (rqc | rqw | rq1)
Data:

Data will return daily historic weather data entires grouped by stations. It includes all of the station information from the specified starting date until the final date. This data is received in json format. If you wish to retrieve data of only one specific date then only specify a starting date.

Parameters Use Default
"station=" + (station id) Brings data of this station all
"elem=" + (prcp | tmax | tmin | tavg) Request the specific information of said station null
"calc=" + (none | avg | max | min | stddev | stderr | all) Performs an operation with the available information none
"startdate=" + ISOdate (YYYY-MM-DD) Is the starting date used to perform the request none
"enddate=" + ISOdate (YYYY-MM-DD) Is the final date used to perform the request startdate

Each parameter takes in values that affect the request's outcome. To help filter out what type of information is requested we created some values that they are expected to take. Each value has the job of specifying a piece of data or specific function the database must perform. Below you will find all the possible values and their use.

Values Use
none Returns values of date interval
avg Averaged values of date interval
max Maximum values of date interval
min Minimum values of date interval
stddev Standard deviation of date interval
stderr Standard error of date interval
all Returns all previous calculations in a single json
prcp Filters for precipitation
tmax Filters for the max temperature
tmin Filters for the minimum temperature
tavg Returns the value of the average temperature

Query Examples:

Example 1:
		 http://climatologia.uprm.edu:8008/api?q=data&elem=prcp&startdate=2018-03-01&enddate=2018-03-31

This query will return all prcp entries from March 1, 2018 to March 31, 2018. Here q was specified to be data, which allowed use of data's additional parameters. The elem was specified to be prcp which defined the element to be precipitation. The starting range startdate was specified to be 2018-03-01 which defined the start of the date interval and the ending range enddate was specified to be 2018-03-31 which defined the end of the date interval. If only one specific date is desired only specify startdate and omit the enddate.

Example 2:
		  http://climatologia.uprm.edu:8008/api?q=data&elem=prcp&startdate=2018-03-01

Since calc was not specified, its default value was none, hence the query returns all recorded dates and their data. Specifying calc (i.e. avg) would result in one average prcp value across the date interval per station.

Example 3:
		  http://climatologia.uprm.edu:8008/api?q=data&elemdata&>=tmax&calc=avg&startdate=2018-03-01&enddate=2018-03-31

This query is asking the data base to bring back data of the maximum temperature. Then it will calculate the average value from the date inverval. This date inteval starts startdate from March 1, 2018 up to the end date enddate of March 31, 2018.
